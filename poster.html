Задачи TI

  1. При анализе собственной инфраструктуры и поиску IoC:
    • TI используется в процессе разведки и сбора данных об уязвимостях и угрозах для получения максимально возможного количества информации о злоумышленниках
    • В качестве аналитической базы знаний для обнаружения и обработке угроз, формированию рекомендаций по реагированию
  2. Для организации работы Security Operations Center (SOC):
    • Повышение качества детектов СЗИ
    • Ускорение принятия решений при разборе инцидентов ИБ благодаря контексту
    • Приоритизация угроз и атак в зависимости от критичности индикаторов

Индикатор компрометации

Объект или артефакт, обнаруженный в ИТ-инфраструктуре компании, наличие которого с высокой долей вероятности может свидетельствовать о готовящейся, совершающейся или уже осуществленной компьютерной атаке

  • хэш-суммы файлов, их имена и расположение
  • IP-адреса
  • DNS-имена серверов в сети Интернет
  • Конкретные URL
  • Названия веток и ключей реестра Windows

Сбор информации

  1. Песочницы
    • Тестирование влияния ВПО и действий атакующих на специально подготовленных средах для выявления слабых точек системы
  2. Ханипоты
    • Подготовка заведомо уязвимой системы в качестве приманки для выявления инструментов и ТТР, которыми пользуются злоумышленники
  3. Анализ тематических форумов (Darknet)
    • Проникновение в закрытые источники, такие как Darknet веб-форумы для извлечения информации, которой делятся злоумышленники на “своих” ресурсах
  4. Материалы расследований
    • Обработка отчетов вендоров о проведенных расследованиях для извлечения информации об индикаторах, артефактах и ТТР атакующих. Также для определения отрасли по которой работают злоумышленники
  5. PoC
    • Уже готовые Proof of Concept материалы на известные CVE, которые можно найти в открытых источниках, например репозитории GitHub. С помощью PoC можно зафиксировать, какие параметры системы жертвы уязвимы и являются целевыми

Применение фидов

  1. Snort
    • Защищает определённый сегмент локальной сети от внешних атак из интернета. Система Snort выполняет протоколирование, анализ, поиск по содержимому, а также широко используется для активного блокирования или пассивного обнаружения целого ряда нападений и зондирований
  2. YARA
    • В роли инструмента выполняет сигнатурный анализ на основе формальных YARA-правил. В них содержатся индикаторы компрометации для разных типов вредоносного ПО. Основная их задача — определять переменные в шаблонах, найденных в образце вредоносного ПО. При полном или частичном совпадении (в зависимости от конкретного правила) этот алгоритм может быть использован для успешной идентификации ВПО
  3. SIGMA
    • “Sigma для логов — это как Snort для трафика или Yara для файлов”. Специально созданный YAML файл, в котором указывается соответсвие используемых в правилах полей реальным данным в бэкенде. Задумывался он в качестве универсального формата описания правил детектирования, основанного на данных из логов

Средства распространения

  1. Форматы фидов

    • Фиды могут распространяться, например, в стандартных форматах JSON, XML, YAML. Преимуществом таких форматов будут являться простота воспрятия, поскольку такие языки являются текстовыми. Однако, последующее использование может быть затруднительным для использования платформой TI

    • На смену стандартным форматам пришли уже известные STIX 2.0 и TAXII, которые способны быть напрямую интегрированы в TIP. Так же большим плюсом будет визуализация, формат STIX позволяет отображать данные в виде графа

  2. Источники

    • Фиды и правила для СЗИ могут распространяться на открытых ресурсах, одними из основных являются общедоступные git-репозитории. Стоит учитывать, что качество таких данных может быть на низком уровне

    • Экспертные данные для TI, распространяемые на коммерческой основе являются более высокоуровневыми. АО “Перспективный мониторинг” предоставляет возможность получать актуальную и проверенную информацию в виде TI feeds, баз решающих правил для СЗИ и агрегированном представлении данных на TI Portal

Threat Intelligence Platform

  • TI-платформа агрегирует знания об угрозах из большого числа внешних источников, нормализует их и выстраивает связи между ними. Далее она распространяет знания об угрозах на используемые средства защиты, повышая их эффективность

    • Открытые платформы агрегации данных TI фактически являются бесплатными, но качество индикаторов компрометации, контекста о ВПО и группировках может оказаться на низком уровне. Таким образом, из-за некачественных данных, организации могут понести весомый ущерб

    • Платформы, разрабатываемые внутри IT-организации, помимо внутреннего использования могут быть распространены в качестве продукта для получения прибыли. Также коммерческие платформы обслуживают команды специалистов, с которыми можно делиться обратной связью с целью повышения качества работы продукта
